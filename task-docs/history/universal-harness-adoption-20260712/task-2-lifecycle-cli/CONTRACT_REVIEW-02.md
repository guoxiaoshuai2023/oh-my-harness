## Findings

### P1 - Drift Versus Incomplete Operation

Valid-state hash mismatch is permitted as confirmable managed-content drift, while a partial update can leave the same old-state/current-byte mismatch and is required to stop as incomplete. No durable observation distinguishes the cases.

Required correction: freeze one observable classification for changed, missing, partially updated, and temporary-residue states, including exact next-invocation behavior.

Calibration: `CAL-004`, `CAL-006`.

### P1 - Git Discovery Is Not Proven Read-Only Or Exact-Surface

Ordinary `git status` can refresh `.git/index`, and directory parent pathspecs expose unrelated descendants.

Required correction: disable optional locks and configured execution surfaces, use literal exact-destination pathspecs only, inspect parents through bounded filesystem operations, and prove Git metadata/unrelated paths were not written or disclosed.

Calibration: `CAL-004`, `CAL-ADAPT-002`.

### P1 - Managed-Block Byte Ownership

The end-marker boundary does not define ownership of the trailing LF or drifted bytes. State stores only a block hash, leaving target suffix classification ambiguous. The lookalike grammar also needs closed token order and boundaries.

Required correction: define the exact owned byte interval for clean and drifted blocks, backup contents, scanner ordering, case handling, token boundaries, and conflict outcomes.

Calibration: `CAL-004`, `CAL-SEM-001`.

### P1 - Package Generation And Cleanup

The package `files` representation does not literally match the accepted Task 1 package contract, cleanup recursively removes `dist` despite narrower authority, temporary-root cleanup is omitted, and prior generated-subtree ownership lacks a test.

Required correction: align package metadata literally; require an absent `dist` before generation; freeze provenance checks and failure-safe cleanup that removes only verified generated bytes and the owned temporary root.

Calibration: `CAL-004`.

### P1 - State Identity And Compatibility

Semver, timestamp, target-root hash, canonical JSON, operation-ID serialization, uninstall version representation, and the package/inventory/state compatibility matrix remain open.

Required correction: freeze exact lexical/canonical encodings, operation-ID preimages for every operation, and compatibility/rejection rules.

Calibration: `CAL-004`, `CAL-ADAPT-003`.
